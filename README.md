# AOS - Advanced Operating System

# RU (Русский)

## Что это?

**AOS (Advanced Operating System)** — это высокопроизводительная многозадачная операционная система для архитектуры **x86_64**, построенная на принципах **гибридного ядра**. Изначально задуманная как микроядерная система, AOS эволюционировала в гибридную модель для достижения максимальной пропускной способности при сохранении модульности сервисов.

## Философия

В основе AOS лежит **концепция минимального вмешательства ядра в работу приложений**, при этом критические компоненты оптимизированы для работы в защищенном режиме. Также AOS **отличается легким IPC**, не компрометирующий безопасность.

В AOS **драйвером является поток**, а не процессом (как в других системах). ОС разделяет драйвера на 2 вида:
- **Демон** - любой поток, зарегистрированный как драйвер для работы с IPC, но не требующий специальных прав.
- **Аппаратный драйвер** - приложение, имеющее особую точку входа и имеющую специальную секцию с информацией о драйвере (в разработке).

Также одна из идей AOS - возможность портирования программ линукса (работа других ОС не гарантируется, даже если все необходимые функции предоставлены). Подробнее здесь -> [AOSLIB-linux](https://github.com/angeloxhek/AOSLIB-linux) <-

AOS имеет независимое API (AOSLIB), обеспечивающее весь функционал системы, в отличие от портов. В портах функционал выходит чуть позже.

AOS имеет специальный драйвер для запуска других различных драйверов (в том числе и от сообщества) - initdriver. Однако имеются обязательные драйвера, которые загружаются напрямую ядром: authdriver и vfsdriver.

AOS в качестве исполняемых файлов поддерживает любые файлы формата ELF64 (ELF32 на данный момент не поддерживается, расширение не обязательно, однако драйвера authdriver, vfsdriver и initdriver имеют особенности в именовании).

## Загрузчик

Структура заголовка файла ядра (некоторые данные могут отличаться):
```
EB 1E 41 4F 53 4C 44 52 A9 33 01 00 00 00 00 00 ë.AOSLDR©3......
01 00 00 00 00 70 00 00 00 00 00 00 90 90 90 90 .....p..........
```
- `EB 1E` (`jmp short $00000020`) - инструкция для пропуска заголовка
- `41 4F 53 4C 44 52` (`AOSLDR`) - байты для проверки валидности файла ядра
- `A9 33 01 00 00 00 00 00` (`0x133A9`) - адрес (в файле) - указатель на основную точку входа `kernel_main`. Все загрузчики (кроме BIOS) обязаны вызывать эту функцию вместо инструкции из п.1. Длина 8 байт, не смотря на то, что используется только 4 (загрузчики обязаны читать 8 байт)
- `01 00 00 00` (`1`) - версия ядра. Загрузчик (кроме BIOS) обязаны проверять это значение на совместимость, не смотря на проверку этого значения ядром. Длина 4 байта
- `00 70 00 00 00 00 00 00` (`0x7000`) - адрес, куда можно гарантированно записывать структуру `boot_info_t`. Значение можно использовать по желанию, однако его использование повышает стабильность системы. Длина 8 байт
- `90 90 90 90` (`90` - иструкция nop) - заглушка, завершающая заголовок. Прыгать сюда небезопасно, т.к. здесь могут оказаться данные. Для правильного прыжка используйте инструкцию из п.1 или адрес из п.3. Длина плавающая, добивает заголовок до кратности 16 байт

На данный момент поддерживаются следующие варианты загрузки системы:

### BIOS (MBR)

На данный момент является основным загрузчиком. Загрузка происходит через PBR, записанный на раздел с форматом FAT32 (см. [здесь](https://github.com/angeloxhek/AOS/main/README.md#%D0%BF%D0%BE%D1%87%D0%B5%D0%BC%D1%83-%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0-%D0%BD%D0%B0%D1%85%D0%BE%D0%B4%D0%B8%D1%82%D1%81%D1%8F-%D0%BD%D0%B0-fat32)). Ядро имеет спциализированную точку входу для систем с BIOS (несовместима с другими загрузчиками). Физический адрес - AOSLDR.BIN+0x0, виртуальный - 0x10000. При написании кастомных загрузчиков для BIOS убедитесь, что ваши адреса совместимы с аресами AOS, иначе система будет работать непредсказуемо.
